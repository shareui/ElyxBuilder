# ElyxBuilder

CLI-инструмент для сборки Elyx-плагинов.

## Установка

```
pip install ElyxBuilder
```

## Требования

- Python >= 3.10
- Python 3.11 — нужен только для компиляции `.pyc`
- pyzipper — нужен только для шифрования архивов

---

## Команды

> Все команды запускаются из корня проекта — директории, в которой находится `refmap.yml`.

### `elyb --version`

Показывает версию ElyxBuilder.

---

### `elyb new`

Создаёт структуру нового плагина в текущей директории.

По умолчанию открывается интерактивный режим: для каждого поля `meta.yml` показывается сгенерированное значение в скобках — нажмите Enter, чтобы принять его.

```bash
elyb new
```

С флагом `-g` / `--gen` пропускает интерактивный режим и создаёт плагин сразу по переданным флагам:

```bash
elyb new -g -n "My Plugin" -a myname
elyb new -g -n "My Plugin" -a myname -zf eaf
```

| Флаг | Описание |
|---|---|
| `-g`, `--gen` | Быстрое создание (без интерактивного режима) |
| `-n`, `--name` | Имя плагина (обязательно с `--gen`) |
| `-a`, `--author` | Идентификатор автора (обязательно с `--gen`) |
| `-zf`, `--zipformat` | Расширение архива (по умолчанию: `eaf`, только с `--gen`) |

Имя нормализуется для файлов/папок: пробелы убираются с переводом следующего слова в CamelCase, спецсимволы (кроме `_`, `-`, букв и цифр) удаляются. В `meta.yml` имя сохраняется как есть.

ID плагина формируется как `author_PluginName`, максимум 32 символа.

`description` всегда генерируется автоматически как плейсхолдер `{description}` и не запрашивается.

### `elyb build`

Собирает плагин в архив. Запускается из директории с `refmap.yml`.

```bash
elyb build
elyb build -v
elyb build --no-assets
elyb build --no-folder
elyb build --ast
elyb build --compile
elyb build --compile 2
elyb build --compile --reset
elyb build --compile -o
elyb build --compile -o src/module.py src/other.py
elyb build -p aes-256 mypassword
elyb build -sv 1.0.0
elyb build -sv 1.0.0 true
elyb build -sc com.example.client
elyb build -sc com.example.client myclient
```

| Флаг | Описание |
|---|---|
| `--no-assets` | Исключить файлы из `optionalAssets` |
| `-nf`, `--no-folder` | Исключить директорию `elyxbuilder` из архива |
| `-v`, `--verbose` | Подробный лог сборки |
| `-a`, `--ast` | Проверить синтаксис `.py` через AST перед сборкой |
| `-c [LEVEL]`, `--compile [LEVEL]` | Скомпилировать `.py` → `.pyc` (Python 3.11); LEVEL — уровень 0–2 (по умолчанию: 1) |
| `-r`, `--reset` | Очистить кэш компиляции перед сборкой (только с `--compile`) |
| `-o [FILE...]`, `--obfuscation [FILE...]` | Обфусцировать исходники перед упаковкой; без файлов — весь source |
| `-p METHOD PASS` | Зашифровать архив |
| `-ni`, `--no-info` | Не добавлять блок с информацией elyxbuilder в `meta.yml` |
| `-sv VERSION [APPEND]` | Добавить `staticVer` в блок информации о сборке; необязательный `APPEND=true` добавляет `-{version}` к имени архива (по умолчанию: `false`) |
| `-sc PACKAGE [NAME]` | Добавить `client` в блок информации о сборке; необязательный `NAME` добавляет `-{name}` к имени архива |

`--ast` и `--compile` взаимоисключающие.

Результат кладётся в `builds/`.

#### Информация о сборке

Перед упаковкой elyxbuilder дописывает блок с комментарием в `meta.yml` внутри архива. Файл на диске не изменяется.

```yaml
# elyxbuilder info
compiled: true/false
buildNum: 5
buildDate: 2026-05-09
pythonVer: 3.11
sourceHash: a3f2...
elybVer: 0.3.0
staticVer: "1.0.0"
client: "com.example.client"
```

`staticVer` присутствует только при передаче `-sv` / `--static-version`. Если необязательный второй аргумент равен `true`, к имени архива добавляется `-{version}` (например, `MyPlugin-1.0.0.eaf`).

`client` присутствует только при передаче `-sc` / `--static-client`. Если указан необязательный второй аргумент, к имени архива добавляется `-{name}` (например, `MyPlugin-myclient.eaf`).

Используйте `-ni` / `--no-info`, чтобы пропустить этот блок.

#### Компиляция (`--compile`)

Файлы из `compilationIgnore` не компилируются и попадают в архив как `.py`. Остальные `.py` заменяются скомпилированными `.pyc`. Используется инкрементальный кэш — повторная сборка перекомпилирует только изменённые файлы.

Необязательный аргумент уровня (0–2) соответствует уровням оптимизации `py_compile`:

| Уровень | Эффект |
|---|---|
| `0` | Без оптимизации (assert'ы и docstring'и сохраняются) |
| `1` (по умолчанию) | Удаляет assert-выражения |
| `2` | Удаляет assert-выражения и docstring'и |

Изменение уровня инвалидирует кэш — все файлы будут перекомпилированы.

#### Шифрование (`-p`)

Требует: `pip install pyzipper`

| Метод | Описание |
|---|---|
| `zipcrypto` | Стандартное ZIP-шифрование |
| `aes-128` | AES 128-bit |
| `aes-192` | AES 192-bit |
| `aes-256` | AES 256-bit (рекомендуется) |

```bash
elyb build -p aes-256 mypassword
```

### `elyb cached`

Показывает, какие файлы изменились с последней компиляции. Запускается из директории с `refmap.yml`.

```bash
elyb cached
```

Требует предварительной сборки с `--compile`.

| Статус | Описание |
|---|---|
| `ok` | Файл не изменён, кэш актуален |
| `modified` | Файл изменился с последней компиляции |
| `new` | Файл ещё не компилировался |
| `ignored` | Файл в `compilationIgnore` |

### `elyb watch <seconds>`

Следит за изменениями в директории плагина и автоматически запускает сборку. Запускается из директории с `refmap.yml`.

```bash
elyb watch 20
elyb watch 20 --args "-c -v -nf -sv 12.6.4 true"
elyb watch 10 -a "-c"
```

| Флаг | Описание |
|---|---|
| `-a`, `--args` | Аргументы для `elyb build` (в кавычках) |

Запускается в интерактивном полноэкранном режиме. Каждые `<seconds>` секунд сканирует директорию плагина на изменения. Если файл был добавлен, удалён или изменён — запускается сборка с переданными аргументами.

Архив всегда сохраняется как `builds/latest.eaf` (или с нужным расширением). Логи сборки не отображаются — на экране видны только события watch.

**Управление:**

| Клавиша | Действие |
|---|---|
| `Q` | Выход |
| `P` | Пауза / Возобновление |

**Сообщения в логе:**

| Сообщение | Когда |
|---|---|
| `• Polling started` | При запуске |
| `• Checking` | Каждый тик проверки |
| `• Nothing has changed` | Изменений нет |
| `• Changes found` | Обнаружены изменения |
| `• Starting the built` | Сборка запускается |
| `• Built successfully: builds/latest.eaf` | Сборка успешна (зелёным) |
| `• Built failed: {ошибка}` | Сборка упала (красным) |
| `• Paused` | Опрос на паузе |
| `• Resumed` | Опрос возобновлён |
| `• Bye-bye` | При выходе |

### `elyb add-ignore <path> <target>`

Добавляет путь в один из списков игнорирования в `.elyxbuilder/config.yml`.

```bash
elyb add-ignore "MyPlugin/res/heavy.png" --no-assets
elyb add-ignore "MyPlugin/.elyxbuilder/cache/*" --all
elyb add-ignore "MyPlugin/src/helpers.py" --compile
```

| Флаг | Список | Эффект |
|---|---|---|
| `-a`, `--all` | `ignoreAll` | Исключить из любой сборки |
| `-na`, `--no-assets` | `optionalAssets` | Исключить при `--no-assets` |
| `-c`, `--compile` | `compilationIgnore` | Не компилировать |

Обратные слеши нормализуются в прямые. Дубликаты не добавляются.

---

### `elyb del-ignore <index> <target>`

Удаляет запись из списка игнорирования по индексу (с нуля).

```bash
elyb del-ignore 0 --all
elyb del-ignore 2 --no-assets
elyb del-ignore 1 --compile
```

Флаги те же, что у `add-ignore`. Индекс соответствует позиции в списке в `config.yml`.

## Обфускация

> **Бета.** Обфусцированные сборки могут приводить к непредвиденному поведению во время выполнения. Тщательно тестируйте перед распространением. Используйте на свой страх и риск.

Обфускация преобразует исходный код Python через AST перед упаковкой. Работает как самостоятельно, так и вместе с `-c`:

```bash
elyb build -o                 # обфусцировать все исходники, включить как .py
elyb build -o src/module.py  # обфусцировать только указанные файлы
elyb build -c -o              # обфусцировать + скомпилировать в .pyc
elyb build -c -o src/a.py    # обфусцировать указанные файлы + скомпилировать
```

Без аргументов-файлов обфусцируются все `.py` из директории `source`. С аргументами — только указанные пути (относительно корня проекта); остальные обрабатываются как обычно.

При использовании с `-c` обфусцированные файлы никогда не кэшируются — каждая сборка компилирует их заново, чтобы гарантировать свежий случайный результат.

После успешной обфусцированной сборки в `builds/latest_mapping.json` сохраняется файл маппинга. В нём зафиксировано, как были переименованы функции и классы верхнего уровня — удобно для отладки.

### Этапы пайплайна

Пайплайн выполняется в следующем порядке. Каждый этап можно отдельно отключить через `config.yml` (см. [Конфиг](#конфиг-обфускации) ниже).

#### 1. Удаление docstring'ов (`stripDocstrings`)

Удаляет все docstring'и из модулей, классов и функций. Первый строковый литерал в каждом теле отбрасывается; если тело становится пустым, вставляется `pass`.

Пропускается для узлов, помеченных `@ELYBNoObf`.

#### 2. Удаление log-вызовов (`removeLogs`)

Удаляет все вызовы `log(...)` как выражения-операторы. Строки, помеченные `# ELYBsaveLog`, сохраняются.

#### 3. Переименование локальных имён (`renameLocals`)

Переименовывает локальные переменные, параметры функций и локальные имена функций/классов в случайные идентификаторы (4–12 символов, например `xKt3p`). Переименование происходит в рамках области видимости и последовательно — одно и то же оригинальное имя всегда получает одно и то же обфусцированное имя внутри одной функции.

Имена, которые никогда не переименовываются:

- `self` и `cls`
- дандер-имена (`__init__`, `__name__` и т.п.)
- имена, импортируемые или экспортируемые между модулями проекта
- имена параметров, используемых как keyword-аргументы где-либо в проекте
- имена, на которые ссылаются `nonlocal` или `global`
- имена внутри узлов, помеченных `@ELYBNoObf`

Для классов, наследующих внешний базовый класс (не определённый в проекте), имена методов и параметров сохраняются — Java/внешний мост обращается к ним по оригинальным именам.

#### 4. Кодирование строк (`encodeStrings`)

Заменяет строковые литералы XOR-выражением для декодирования:

```python
# оригинал
x = "hello"

# после обфускации
x = bytes(b ^ 42 for b in b'\x66\x4f\x46\x46\x45').decode()
```

XOR-ключ — случайный байт, выбирается заново при каждой сборке. Import-операторы не затрагиваются. Строки, помеченные `# ELYBnoStrobf`, пропускаются. F-строки сохраняются как есть (извлекаются до пайплайна, восстанавливаются после).

#### 5. Кодирование чисел (`encodeNumbers`)

Заменяет целочисленные литералы XOR-выражением со случайной 16-битной маской:

```python
# оригинал
x = 1000

# после обфускации
x = 27736 ^ 26792
```

Тривиальные значения `0`, `1` и `-1` не кодируются — они слишком распространены, а шум не даёт практической пользы. Булевы значения не затрагиваются никогда. Строки, помеченные `# ELYBnoIntObf`, пропускаются.

#### 6. zlib-сжатие (`zlibCompression`)

Финальный этап. Сжимает обфусцированный исходник через zlib (уровень 9), кодирует в base64, разворачивает байты и оборачивает весь файл в двухстрочный exec-лаунчер:

```python
_ = lambda __ : __import__('zlib').decompress(__import__('base64').b64decode(__[::-1]))
exec((_)(b'...'), globals(), locals())
```

Payload выполняется с явно переданными `globals()` и `locals()`, чтобы пространство имён модуля корректно работало в рантайме (Chaquopy 17.0.1 / Python 3.11).

Этот этап запускается после всех AST-проходов и работает на уже обфусцированном тексте. По умолчанию отключён (`zlibCompression: false`) — сжатый вывод нельзя скомпилировать в `.pyc`, не используйте вместе с `--compile`.

### Маркеры в исходниках

Маркеры — это встроенные комментарии, управляющие обфускацией на уровне строки или узла.

| Маркер | Область | Эффект |
|---|---|---|
| `# ELYBsaveLog` | строка | Сохраняет вызов `log(...)` на этой строке |
| `# ELYBnoStrobf` | строка | Пропускает кодирование строк на этой строке |
| `# ELYBnoIntObf` | строка | Пропускает кодирование чисел на этой строке |

### Декоратор `@ELYBNoObf`

Примените `@ELYBNoObf` к функции или классу, чтобы полностью исключить его из обфускации. Сам декоратор вырезается из вывода — в скомпилированный архив он не попадает.

```python
@ELYBNoObf
def myHandler(event, data):
    # эта функция не обфусцируется: нет переименований, нет кодирования строк/чисел
    log("event received")  # ELYBsaveLog
```

`@ELYBNoObf` на классе — исключает весь класс. `@ELYBNoObf` на методе — только этот метод.

### Конфиг обфускации

Поведение обфускации настраивается в `.elyxbuilder/config.yml` в ключе `obfuscation`:

```yaml
obfuscation:
  stripDocstrings: true
  removeLogs: true
  renameLocals: true
  encodeStrings: true
  encodeNumbers: true
  zlibCompression: false
```

Все ключи необязательны. По умолчанию каждый равен `true`, кроме `zlibCompression` — он по умолчанию `false`. Установите `false`, чтобы отключить соответствующий этап.

Настройка `removeLogs` также применяется к обычным (не обфусцированным) сборкам — log-вызовы вырезаются из `.py`-файлов, которые попадают в архив как исходники.

> **Примечание:** `zlibCompression` несовместим с `--compile`. При включении вывод — это обычный `.py`-лаунчер, который нельзя скомпилировать в `.pyc`.
